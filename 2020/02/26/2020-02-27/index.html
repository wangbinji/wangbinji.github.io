在一个Objective-C项目中实现与Swift混编

在一个Objective-C项目中第一次创建一个Swift文件的时候，Xcode会提示是否需要创建一个桥接头文件，点击是之后Xcode会自动创建一个桥接文件，文件名为xxxx-Bridging-Header.h

1、Objective-C文件调用Swift文件的属性和方法
a、需要在暴露出来的方法和属性上加@objc，否则不可用；且文件的类需要继承自NSObject或NSObject的子类
b、在Objective-C文件里 #import “xxxx-Swift.h” （xxxx为项目名）
2、Swift文件调用Objective-C文件中的属性和方法
a、需要在统一的bridge头文件里面import相关的Objective-C文件，然后即可使用。
3、Objective-C Generated interface Header Name
a、Targets -> Build Settings -> Swift Compiler -> General -> Objective-C Generated Interface Header Name
进行配置、默认文件名是 工程名-Swift.h
4、使用第三方framework
a、Target -> Build Setting -> Packaging -> Defines Modules YES
b、Target -> Build Phases -> Link Binary 添加要导入的framework
c、配置桥接文件，比如要使用abc-lib.framework库中的abc.h #import “abc-lib/abc.h”
